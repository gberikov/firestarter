# Дизайн: отмена обратного отсчёта при запуске

Дата: 2026-05-23

## Проблема

Обратный отсчёт в `launch_task` существует именно для того, чтобы оператор мог
отменить запуск ракеты, пока идёт счёт. Сейчас такой возможности нет:
`countdown_beeps()` ([main/firestarter.c](../../../main/firestarter.c)) — это
блокирующий цикл из вызовов `beep()` и `vTaskDelay()`, который всегда доходит до
конца и подаёт питание на `MOSFET_GPIO`. Точки выхода нет.

## Цель

Дать возможность прервать запуск из веб-интерфейса в течение обратного отсчёта.
При успешной отмене: немедленно остановить зуммер, показать «Запуск отменён»,
вернуть кнопку в исходное состояние. **MOSFET не должен сработать.**

## Подход

Выбран **опрашиваемый флаг отмены** (polled abort flag). HTTP-запрос `/cancel`
выполняется в потоке httpd, а отсчёт идёт в задаче `launch_task`, поэтому отмена
сводится к установке флага, который задача отсчёта проверяет сама. Задача
`launch_task` остаётся единоличным владельцем всех изменений GPIO и зуммера —
ничего не убивается между потоками (никаких `vTaskDelete` извне, никаких
notification-примитивов).

Граница окна отмены: отмена действует только на 9 шагах счёта. Как только
отправлено сообщение `launch` («ЗАПУСК!»), запуск **зафиксирован** — финальный
2-секундный тон и импульс MOSFET уже не отменяются.

## Изменения в прошивке (`main/firestarter.c`)

### Новое состояние

```c
static volatile bool launch_abort = false;
```

### Помощник отменяемой задержки

Спит срезами по 50 мс, чтобы отсчёт мог прерваться в середине ожидания:

```c
// возвращает true, если во время ожидания запросили отмену
static bool cancellable_delay_ms(int ms);
```

Худший случай задержки реакции на отмену ≈ один срез 50 мс.

### `beep()` — учитывает отмену

Ожидание длительности тона идёт через `cancellable_delay_ms`. При отмене функция
обнуляет скважность LEDC (заглушает зуммер) и возвращает `true`. Сигнатура
меняется на возврат `bool`. `startup_melody()` продолжает вызывать `beep()` и
просто игнорирует возвращаемое значение — вне запуска `launch_abort` всегда
`false`, поэтому стартовая мелодия не меняется.

### `countdown_beeps()` — возвращает `bool`

`true` = отменено. Проверяет флаг в начале каждого из 9 шагов и пробрасывает
отмену из каждого `beep()` / задержки. После отправки сообщения `launch`
финальный тон `beep(1500, 2000)` не отменяется.

### `launch_task()`

В начале сбрасывает `launch_abort = false`, затем:

- если `countdown_beeps()` вернул «отменено» → лог «Запуск отменён»,
  широковещательно `{"type":"cancelled","message":"Запуск отменён"}`, очистка
  (`launch_in_progress = false`, `launch_task_handle = NULL`) и `vTaskDelete`
  **без единого обращения к MOSFET**;
- иначе → импульс MOSFET ровно как сейчас.

### Новый обработчик `/cancel`

Регистрируется как `httpd_uri_t` в `start_webserver()` рядом с `/launch`:

```c
// устанавливает launch_abort = true, только если запуск идёт
GET /cancel  ->  "Отмена запущена" / "Нет активного запуска"
```

## Изменения в UI (`main/index.html`)

Кнопка запуска получает режим (`idle` → `cancel` → `committed`). Один обработчик
клика выбирает действие по текущему режиму (`launchRocket()` или
`cancelLaunch()`).

| Принятое сообщение | Кнопка |
|---|---|
| `countdown_start` / `countdown` | **«Отмена»**, красная, активна → клик шлёт `GET /cancel` |
| `launch` («ЗАПУСК!») | неактивна (зафиксировано, окно отмены закрыто) |
| `cancelled` (новое) | назад в **«Запуск»**, оранжевая, активна; счётчик очищен; показывает «Запуск отменён» |
| `status` (пуск выполнен) | назад в **«Запуск»** как сейчас |

- Новый стиль `.cancel-btn` (красный).
- Новый `case 'cancelled'` в `handleMessage()`.
- `cancelLaunch()` делает `fetch('/cancel', { method: 'GET', credentials: 'omit' })`.

## Соглашения и проверка

- Все новые пользовательские строки и логи — на русском, по конвенции проекта.
- Автотестов нет: проверка на железе через `idf.py monitor` (тег `firestarter`)
  и веб-интерфейс. Перед проверкой сборка `idf.py build` должна пройти.

### Сценарии проверки

1. Запуск → отмена на середине счёта: зуммер замолкает сразу, появляется «Запуск
   отменён», кнопка снова «Запуск», MOSFET не сработал (нет лога «Пуск выполнен»).
2. Запуск без отмены: счёт доходит до конца, MOSFET срабатывает как раньше.
3. `/cancel` без активного запуска: ответ «Нет активного запуска», ничего не
   происходит.
4. Отмена в момент «ЗАПУСК!» (уже зафиксировано): запуск всё равно выполняется.
